# Event 3: Cloud Architect Final & AWS Meetup 11/07/2026

### Event Information

| Item | Details |
|---|---|
| **Event name** | Cloud Architect Final & AWS Meetup 11/07/2026 |
| **Time** | 09:00  11/07/2026 |
| **Location** | 26th Floor, Bitexco Tower, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City |
| **Role** | Attendee |

### Event Objectives

- Attend the **Cloud Architect** final round between **KLKAT** and **Ngu Dai Hiep**.
- Learn how teams analyze requirements, design cloud architectures, and present technical solutions.
- Listen to technical sharing sessions about web application security, SLA/monitoring, and AWS Cloud Practitioner exam preparation.

### Main Content

#### 1. Opening Session: Cloud Architect Final

The event started with the **Cloud Architect** final round between **KLKAT** and **Ngu Dai Hiep**. This session helped me observe how teams approached a problem, selected suitable AWS services, built a cloud architecture, and defended their solution in front of the judges.

From this session, I learned that a Cloud Architect needs more than AWS service knowledge. The role also requires system thinking and the ability to balance security, cost, performance, availability, and operational readiness.

#### 2. Thinh Nguyen - Securing Your Web Apps With AWS Security Agent

The session by **Thinh Nguyen** focused on securing web applications deployed on AWS. It helped me understand common risks in web applications, the need for layered protection, and the importance of proactively detecting and preventing suspicious access patterns.

From this talk, I learned that securing a cloud-based website is not only about opening the right ports or configuring Security Groups. It also requires monitoring traffic behavior, controlling abnormal requests, and preparing protection mechanisms early.

#### 3. Nguyen Huynh Son - SLA and Monitoring: From SLA to Monitoring What Really Matters

The session by **Nguyen Huynh Son** explained the relationship between **SLA** and **Monitoring**. The key message was that monitoring should not only check whether a system is running, but should focus on the metrics that truly affect user experience and service quality commitments.

This session helped me understand the importance of tracking metrics such as availability, latency, error rate, and early warning signals when a system starts behaving abnormally. These are important skills for operating real cloud systems.

#### 4. Ngo Le Tan Huy - Inside The Exam: AWS Cloud Practitioner

The final session by **Ngo Le Tan Huy** shared insights about the **AWS Certified Cloud Practitioner** exam. The content helped me understand the exam structure, the scope of knowledge to prepare, and a suitable learning approach for beginners.

This sharing helped me organize my learning path more clearly. Before moving to advanced certifications, I need to strengthen my understanding of core AWS topics such as compute, storage, database, networking, security, billing, and cloud concepts.

### What I Learned

- Gained a clearer understanding of how to present and defend a cloud architecture.
- Learned the importance of web application security when deploying systems on AWS.
- Understood that monitoring should be connected to SLA and user-impacting metrics.
- Gained more direction for studying and preparing for the AWS Cloud Practitioner exam.

### Relation to My Internship Project

After this event, I gained practical insights that can be applied to the **SUMMER-STORE** project. When deploying the website to AWS, I should pay more attention to API security, Security Group configuration, backend logs, EC2-to-RDS connectivity, and operational metrics instead of only checking whether the website can be accessed.

### Results Achieved

Event 3 helped me better understand cloud architecture thinking, web application security, SLA-based monitoring, and the AWS certification learning path. It was a useful meetup that gave me practical knowledge to improve how I deploy and operate projects on AWS.
